# 인터엑스 프론트엔드 과제 – 채용 관리 보드

지원자들을 단계별로 관리할 수 있는 **칸반(Kanban) 형태의 채용 관리 보드**를 구현했습니다.  
드래그 앤 드롭을 통해 지원자의 상태를 변경하고, 많은 지원자가 존재하더라도 안정적인 성능과 UX를 유지하는 것을 목표로 설계했습니다.

## 기술 스택
- **React 18**
- **TypeScript**
- **Vite**
- **Tailwind CSS**
- **@dnd-kit**
- **@tanstack/react-virtual**
- **React Router**
- **LocalStorage**
- **GitHub Actions (CI)**
- **Husky / lint-staged**

## 요구사항 및 구현 내용

### 드래그앤 드롭 기능
- 지원자 카드는 **드래그 앤 드롭**을 통해 단계 간 이동이 가능합니다.
- `@dnd-kit` 라이브러리를 사용해 구현했습니다.

> dnd-kit 라이브러리 선택 이유
> - 상하/좌우 모두 자유롭게 이동 가능
> - 리스트, 그리드, 캔버스 등 다양한 레이아웃에 적합
> - 컴포넌트 기반이 아닌 hook 중심의 설계
> - 높은 유연성과 커스터마이징 가능성

- 초기에는 onDragEnd만 사용해 드롭 시점에만 상태를 변경했는데, 이 경우 드래그 중 위치 미리보기가 없어 UX가 끊기는 문제가 있었습니다. 이를 해결하기 위해 onDragOver에서 임시 상태를 업데이트하고, 드롭 시점에만 최종 확정 및 저장하는 구조로 개선했습니다.

- 초기 Drag & Drop 로직을 컴포넌트 내부에서 단일 파일로 관리했지만, 리팩토링을 통해 UI 이벤트 처리와 카드 이동 규칙을 분리했습니다. 카드 이동을 순수 함수 기반의 도메인 로직으로 추출하고, DragOver와 DragEnd가 동일한 이동 규칙을 공유하도록 구조를 개선했습니다. 이를 통해 가독성과 유지보수성을 높이고, order 정합성을 단일 지점에서 관리할 수 있도록 설계했습니다.

### 스크롤 기능 및 성능 최적화
- 각 채용 단계 컬럼에는 스크롤이 적용되어 있습니다.
- 지원자가 많아질 경우를 대비해 `@tanstack/react-virtual`을 사용해 카드 리스트 가상화를 적용했습니다.
- 이를 통해 렌더링 비용을 최소화하고, 드래그 앤 드롭 + 스크롤 환경에서도 안정적인 성능 유지하도록 했습니다.
  
### 삭제 버튼 기능 
- 각 단계의 지원자 카드에는 삭제 버튼이 함께 표시됩니다.
- 삭제 버튼을 누르면 해당 지원자는 보드에서 제거됩니다.
- 삭제 전에는 확인 모달을 노출해 실수로 인한 삭제를 방지했습니다.

### 채용사이트 이동
- 카드 hover 시 **상세 이동 링크**가 노출됩니다.
- 링크 클릭 시 해당 지원자의 채용 사이트(상세 페이지)로 이동하도록 구현했습니다.
- - 카드 클릭/드래그와 링크 클릭이 충돌하지 않도록 이벤트 처리를 분리했습니다.
 
## 데이터 관리 방식

- 별도의 서버 없이 **LocalStorage 기반 상태 관리**로 구현했습니다.
- 초기 진입 시:
  - LocalStorage에 데이터가 없는 경우에만 mock 데이터를 초기화
- 이후:
  - 드래그, 삭제 등 모든 변경 사항은 LocalStorage에 즉시 반영
- 새로고침 시에도 동일한 상태를 유지할 수 있도록 설계했습니다.


## CI / 개발 환경 설정

### GitHub Actions

- GitHub Actions를 사용해 **CI 파이프라인**을 구성했습니다.
- Pull Request 및 Push 시:
  - 프로젝트 빌드 검증
  - 기본적인 코드 안정성 확인

### Husky & lint-staged

- Husky를 사용해 **Git Hook**을 설정했습니다.
- 커밋 시:
  - lint-staged를 통해 변경된 파일만 검사
  - 코드 품질을 사전에 검증해 불필요한 오류 커밋을 방지했습니다.
